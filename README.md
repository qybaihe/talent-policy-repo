# 人才引进政策调研仓库

聚焦重点城市的人才引进、落户、补贴、安居与购房支持政策整理。这个仓库的目标不是“堆链接”，而是把政策改写成后续可持续维护的**结构化速查稿**：

> 满足什么条件 → 可以获得什么优势 → 哪些点已坐实 → 哪些点仍待复核

适合继续接手、补证据、扩城市。

## 2. 本项目的整理原则

### 2.1 写法原则
统一按下面顺序写：
1. **一句话判断**：这个城市的人才政策更适合怎么理解；
2. **满足什么条件 → 可以获得什么优势**：按学历型、博士型、技能型、应届/青年人才型等拆开；
3. **速查表**：学历、年龄、社保、户籍、职称/技能、留学、核心优势；
4. **来源链接**：优先放官方原始链接；
5. **更新时间**；
6. **待复核**：明确哪些已经坐实，哪些还不能写死。

### 2.2 表达原则
- 尽量避免“政策原文堆砌”；
- 尽量避免把不同通道混写成一个条件包；
- 看到区级差异明显时，不要强行总结成“全市统一标准”；
- 金额、年龄、社保月数、政策有效期，只要没坐实，就明确写“待复核”；
- 历史政策如果有参考价值，可以保留，但必须标出：**旧口径 / 已失效 / 仅供参考**。

## 3. 信息来源优先级

整理时按下面优先级取证：

### A. 第一优先：官方办事指南 / 官方政策原文
优先使用：
- 省级或市级政务服务网办事指南；
- 市政府官网政策文件；
- 人社局、公安局、住建局、公积金中心等主管部门官网；
- 政策实施办法、通知、问答、解读。

这类来源最适合确认：
- 受理条件；
- 年龄限制；
- 社保连续缴纳月数；
- 是否要求本地户籍；
- 办理时限、办理方式；
- 政策是否仍有效。

### B. 第二优先：官方业务知识库 / 政策问答 / 在线咨询答复
适合确认：
- 申报入口；
- 哪个部门负责；
- 区级政策去哪里查；
- 实操解释口径。

但要注意：
- 问答页常能补全“怎么申请”，
- 但不一定足以单独坐实金额和长期有效性。

### C. 第三优先：区级官网、区级人社局、区级人才专区
适合确认：
- 本科/硕士区级补贴；
- 购房、租房、人才公寓；
- 区级社保月数；
- 是否要求单位注册地在本区。

### D. 不建议直接落笔为结论的来源
- 中介、自媒体、公众号二手整理；
- 招聘平台转载；
- 无发布日期或无主管单位的信息页。

这些来源最多只用于“发现线索”，不能直接当最终依据。

## 4. 字段规范

每个城市文档尽量覆盖以下字段：

| 字段 | 含义 | 备注 |
|---|---|---|
| 学历 | 大专、本科、硕士、博士等 | 注意区分全日制、普通高等教育、留学认证 |
| 年龄 | 年龄上限 | 按具体通道拆，不要混写 |
| 社保 | 是否要求在当地缴纳、连续月数 | 尤其注意“累计/连续/补缴是否算” |
| 户籍 | 是否要求本地户籍 | 落户通道和补贴通道常不一样 |
| 职称/技能 | 中高级职称、技师、高级工等 | 是否要求紧缺目录、是否要测试 |
| 留学 | 留学回国、港澳台、外籍人才 | 看是否可免户籍或跨境通办 |
| 安居支持 | 租房补贴、人才公寓、生活补贴 | 与购房支持分开写 |
| 购房支持 | 购房补贴、安家补贴、公积金倾斜 | 没有现行依据不要硬写 |
| 政策有效期 | 文件发布日期、实施日期、是否失效 | 很关键，优先补全 |
| 来源链接 | 官方链接 | 争取一条结论对应一条证据 |

### 额外规范
- **“已坐实”**：已有明确官方页面支撑；
- **“待复核”**：线索存在，但证据还不够；
- **“旧口径”**：政策曾存在，但已失效或现行状态不明；
- **“区级差异”**：必须单独点名，不要模糊写成“各区略有不同”。

## 6. 推荐整理流程

后续继续做时，建议按下面的顺序：

### 第一步：先找“总入口”
先定位：
- 政务服务网办事指南；
- 市政府政策文件库；
- 人社局/公安局/住建局/公积金中心专题页。

目标是先判断这座城市到底属于哪种类型：
- **落户主导型**；
- **补贴主导型**；
- **区级加码型**；
- **安居支持强但统一口径弱**。

### 第二步：拆通道，不混写
至少拆开：
- 学历人才；
- 博士/高层次人才；
- 技能/职称人才；
- 应届/青年人才；
- 留学/港澳台人才。

### 第三步：优先打掉 5 类高价值待复核项
1. **区级补贴金额**；
2. **落户细则**；
3. **社保月数**；
4. **安居/购房支持**；
5. **政策有效期**。

### 第四步：最后再写结论
不要先写判断再找证据。先把官方口径抓实，再写“一句话判断”。

## 8. 推荐下一步优先级

如果下一位接手者只有半天时间，建议按这个顺序推进：

### 优先级 1：深圳
原因：
- 市级通道已经很清楚；
- 剩下的主要是各区补贴信息，属于高确定性增量；
- 做完后能很快形成“最可用”的样板城市。

### 优先级 2：珠海
原因：
- 引进通道已经坐实；
- 最大价值在于确认现行补贴有没有更新版；
- 一旦找到有效新文件，珠海页质量会直接跃升。

### 优先级 3：苏州
原因：
- 信息量很大，区级分散；
- 适合系统性分区推进，不适合一次性想写完；
- 更像一个“小项目”，不是一个补丁。

## 9. 接手提醒

后续接手时，最容易犯的错误有 4 个：
1. 把**旧政策金额**直接当现行标准；
2. 把**区级政策**写成全市统一；
3. 把**落户条件**和**补贴条件**混成一套；
4. 只看新闻稿，不回到**办事指南/实施办法**。

这个仓库当前最重要的不是“再多写几个城市”，而是把重点城市的**证据密度**做高。
